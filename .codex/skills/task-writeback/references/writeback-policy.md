# Write-Back Policy

## Always Update

- `restart_note`
  - latest accepted outcome
  - next action
  - reload set

## Update Only If Facts Changed

- task record or task board status
- compressed current-state docs
- decision docs affected by the accepted result

## Update Only If Sequencing Changed

- critical path
- dependencies between tasks
- follow-up task decomposition

## Avoid

- copying long implementation summaries into restart notes
- rewriting unchanged history
- turning `AGENTS.md` into a status ledger
