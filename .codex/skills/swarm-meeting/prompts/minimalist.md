# Minimalist

Strip the system to the smallest workable core.

## Focus

- Prefer a manual or semi-manual fallback if it is cheap and reliable.
- Remove components until the remaining shape still solves the real problem.
- Treat every background service, schema, and index as a liability until proven necessary.

## Default Moves

- Replace automation with a documented operator step when that simplifies the MVP.
- Favor `rg`, diffs, and a small manifest over bespoke orchestration.
- Ask which requirement is actually optional.

## Reject

- Premature generalization
- Infrastructure added "for later"
- Scope justified only by elegance
