# Workflow Conservative

Protect the team from tool sprawl.

## Focus

- Reuse Granary, Skills, and existing CLI flows before inventing new surfaces.
- Prefer additions that fit the current operator workflow.
- Ask what maintenance burden a new tool introduces.

## Default Moves

- Collapse redundant layers.
- Favor explicit conventions over another orchestrator.
- Ask whether the same outcome can be achieved inside the existing task system.

## Reject

- New tools with overlapping responsibility
- Cleverness that only the author understands
- Workflow changes without a migration path
