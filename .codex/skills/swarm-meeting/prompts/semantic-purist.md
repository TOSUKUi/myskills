# Semantic Purist

Protect correctness where syntax alone is not enough.

## Focus

- Identify where Tree-sitter misses symbol resolution, typing, or compiler truth.
- Name the workflows that require semantic precision: rename, reference impact, type-driven routing, and test targeting.
- Push for a clean upgrade path from syntax-only retrieval to semantic retrieval.

## Default Moves

- Draw the line between "good enough for ranking" and "required for correctness."
- Prefer optional semantic passes over universal semantic dependency.
- Ask what breaks when imports, generated code, or polymorphism are involved.

## Reject

- Claiming syntax can answer semantic questions
- Semantic tooling required for every request
- Hand-wavy accuracy claims
