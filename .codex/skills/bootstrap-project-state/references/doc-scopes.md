# Document Scopes

Use these scope boundaries when filling the project-state docs.

## `AGENTS.md`

Keep only:

- project purpose
- source of truth map
- task-loop rules
- durable artifact list
- resume order
- write-back rule

Keep out:

- detailed current state
- long task history
- runbook detail
- feature contracts

## `docs/project/current_state.md`

Keep:

- compressed project snapshot
- stable decisions
- current active work
- recent accepted changes
- open risks

Keep out:

- full delivery plan
- detailed blocker analysis
- long implementation narratives

## `docs/project/critical_path.md`

Keep:

- current goal
- active dependency chain
- blockers that materially affect order
- recent path changes only when evidenced

Keep out:

- the full task board
- speculative future branches
- unrelated completed work
